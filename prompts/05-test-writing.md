## 📌 填写示例

### 被测目标
- **文件 / 模块**：`hooks/search/useSearchData.ts`
- **功能简述**：搜索输入框的防抖请求 hook，负责调用搜索 API 并管理 loading / results / error 状态

### 测试类型
- [x] 单元测试（hook）
- [x] 组件测试（搜索输入框组件）

### 关键测试场景

#### 正常场景
- [x] 输入关键词后，防抖 300ms 后发起请求，返回搜索结果
- [x] 快速连续输入，只触发最后一次请求（防抖生效）
- [x] 搜索结果为 0 条时，展示空状态提示

#### 边界情况
- [x] 输入为空字符串时，不发起请求
- [x] 关键词含特殊字符（`%`, `#`, 空格）时 URL 正确编码
- [x] 搜索长度超长的关键词（100+ 字符）

#### 异常场景
- [x] 网络请求超时，展示超时错误提示
- [x] 服务器返回 500，展示服务错误提示
- [x] 搜索请求被取消（组件卸载），不触发状态更新

### Mock 策略

| 依赖 | Mock 方式 | 返回值 |
|------|----------|--------|
| `lib/api/search.ts` | `vi.mock` | 模拟搜索结果数组 / 错误抛出 |
| `useDebounce` | 直接测试真实 hook | — |

### 覆盖率目标
- 语句覆盖：90%
- 分支覆盖：85%
