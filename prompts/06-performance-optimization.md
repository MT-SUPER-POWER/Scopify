## 📌 填写示例

### 性能瓶颈描述
歌单页（`/playlist?id=xxx`）加载超过 200 首曲目时，页面滚动有明显卡顿，首次渲染时间约 1.5s。瓶颈在 TracklistTable 组件——全部曲目一次性渲染 DOM，无虚拟化。

### 基准数据（优化前）

| 指标 | 优化前 | 目标 |
|------|--------|------|
| 首次内容渲染 (FCP) | 1.5s | < 800ms |
| 滚动帧率 (200 条) | 20fps | 55fps+ |
| DOM 节点数 | ~3200 | < 500 |
| 内存占用 | ~85MB | < 50MB |

### 优化方向

1. **虚拟滚动**：用 `react-virtuoso` 替换直接 map 渲染，只渲染可视区域 ~20 行
2. **行组件 memo**：`TrackRow` 组件用 `React.memo` + 稳定 key 减少重渲染
3. **图片懒加载**：封面图改用 `loading="lazy"` + 缩略图 URL
4. **合并状态更新**：播放状态变更时不触发整表重渲染

### 影响范围
- `components/Playlist/TracklistTable.tsx` — 核心改动，引入虚拟滚动
- `components/Playlist/TrackRow.tsx` — memo 包裹
- 无后端变更，纯前端优化
- 回退方式：git revert 单个 commit
